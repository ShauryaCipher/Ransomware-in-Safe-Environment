About This Tool

Educational Purpose

This Ransomware Simulation Tool was created purely for educational and cybersecurity awareness purposes. It demonstrates the basic mechanisms of ransomware attacks in a controlled, safe environment to help security professionals, IT personnel, and students understand how these attacks work.

Important: This tool should never be used outside of educational contexts or controlled environments. Using ransomware techniques against real systems without authorization is illegal and unethical.

How This Simulator Works

This simulator uses legitimate encryption technologies, but implements them in a controlled way that:

  • Only operates on files you specifically upload
  • Securely stores the decryption key in your session
  • Provides guaranteed recovery methods
  • Includes detailed educational content
  • Cannot spread to other systems

Technical Details

This simulation uses the Fernet symmetric encryption algorithm from the Python cryptography library. Fernet guarantees that a message encrypted using it cannot be manipulated or read without the key.

Fernet is an implementation of symmetric (also known as "secret key") authenticated cryptography. It uses 128-bit AES in CBC mode and PKCS7 padding, with HMAC using SHA256 for authentication.

Real ransomware often uses a combination of symmetric and asymmetric encryption:

  • Symmetric encryption (like AES-256) for the files themselves
  • Asymmetric encryption (like RSA-2048) for the symmetric key

This simulation only affects files you explicitly upload. It stores them in a temporary directory that is isolated from your system.

In contrast, real ransomware typically:

  • Searches for valuable files throughout the system
  • Targets specific file extensions (documents, images, databases)
  • May attack network shares and connected drives
  • Often deletes shadow copies and backups
  • Can spread laterally through a network

This simulation includes several safety measures:

  • Clear warnings and consent requirements
  • Isolation in a temporary directory
  • Automatic cleanup after simulation
  • Guaranteed recovery through stored keys
  • No persistence or spreading mechanisms
  • Detailed logging of all operations

Best Practices for Prevention

For Individuals
  • Keep software and systems updated
  • Use reputable security software
  • Create regular backups of important data
  • Be cautious with email attachments and links
  • Use strong, unique passwords and 2FA
For Organizations
  • Implement comprehensive backup strategy
  • Train employees on security awareness
  • Segment networks to limit spread
  • Implement principle of least privilege
  • Develop incident response plan
